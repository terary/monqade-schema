Tutorial: MonqadePath

MonqadePath

Overview

There is no class 'MonqadePath' it is an informal extension to Mongoose.SchemaType. When defining the schemaType simply add the additional properties. 


        [pathName]: {
            name: [pathName same as key to this object],
            type: [standard mongoose type],
            isSearchable: [true|false],
            isProjectable: [true|false],
            isUpdatable: [true|false],
            isInsertable: [true|false],
            isRequired: [true|false],
            makeTestData: [function to create test data],
            notes: {
                purpose: 'natural language' ,
                restriction: 'natural language'
            }
        };

MonqadePath is a few properties appended to Mongoose.schema.paths[pathName].options
These properties include:

  • isUpdatable: boolean
  • isInsertable : boolean
  • isSearchable: boolean
  • isProjectable: boolean
  • isRequired: boolean
  • isSystem: boolean
  • makeTestData: function()
  • notes:{purpose:..., restriction:...}
.

isUpdatable

Determines if the path will be read-in by .doUpdateOne(...)

Monqade assumes all pre-existing documents with pre-exisiting path meet the requirements specified in the schema.

Basically, Monqade does not attempt to implement the is[Property] retroactively. All pre-existing documents should be made to meet the specification outlined within the schema..

isInsertable

Determines if the path will be read-in by .doInsertOne(...)
isInsertable default required comments
true true true No issues
true true false No issues
true false true Logically sound.
Empty values my result in end-user errors
true false false Logically sound.
Default values encouraged.
false true true No issues
false true false No issue, required is implicit.
false false true Issue
Code will need to populate.
This is same logic for systemPaths.
false false false No issues

isSearchable

if MonqadeSchema.useRestrictiveSearch=true (default) MonqadeSchema prevents this field from becoming search criteria.

This can be useful for preventing searches for sensitive data or full dataset. Eg: {emails :RegEx(/@/)}

Not full-proof security or even secure - but useful.

isProjectable

If set false MonqadeSchema will prevent this path from becoming an output field/column.

Default projections: The paths that isProjectable=true + paths that are isSystem=true
Projections can also be any subset of paths that are isProjectable=true

At the time of this writing the isProjectable property applies only to doFindMany() and doQueryMany()... All other do[Method]One(...) return full data (exception being doDeleteOne).

isRequired

Pseudo-alias for 'require'

effective required:
required= required && isRequired

isRequired required path will be
true true required
true false required
false true required
false false not required
*pseudo alias: meaning the underlying 'required' field is authoritative and Mongoose/Mongo use it. Monqade, implements isRequired to fit the is[Property] pattern and sets 'required' accord to the above logic.

isSystem

This a derived or virtual. These are fields that:
  1. Added by Mongo
  2. Required by Monqade
All isSystem paths are required for update/delete.

makeTestData

makeTestData: ()=>{return /*something random and datatype appropriate*/}
This is called when creating test records. This should create data sufficiently random to be useful.

notes


            notes: {
                "purpose": "[what is the purpose of this path]",
                "restriction": "[implemented constrains - minimum length 3 characters, not before today, etc]"
              }

This is used for documentation of the schema. Not only in the schema definition but also auto generating API documentation.

Both purpose and restriction should be natural language.
